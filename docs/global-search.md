# 跨库统一搜索

## 入口与交互

- 侧边栏原搜索入口和 `Ctrl+K` 打开全局搜索，不增加第二套入口。
- 输入框聚焦且关键词为空时显示最近搜索；开始输入后立即隐藏历史下拉列表。空关键词下仍保留“新聊天”和最近聊天。
- 搜索历史只记录至少命中一类内容的关键词；加载失败或五类结果总数为零时不写入历史。
- 输入关键词后显示“全部、聊天、图片、素材、灵感、提示词表单”胶囊 Tab，各 Tab 展示当前关键词对应的结果数量。
- “全部”每组默认展示 5 条；点击分类 Tab 或分组中的“查看全部”，会在当前弹窗内加载该类最多 80 条结果。
- 提示词表单结果使用表单自身配置的图标，标题下只显示表单描述，不展示内部分类值。
- 分类滑块按实际按钮宽度平滑移动，支持左右方向键、`Home`、`End` 切换；窄屏下 Tab 横向滚动。
- 方向键切换结果，`Enter` 打开，`Esc` 关闭。输入请求防抖 250ms，过期请求会被取消。

## 搜索接口

```http
GET /api/search?q=<keyword>&scope=<scope>&limit=<limit>&offset=<offset>
```

`scope` 支持 `all`、`chat`、`images`、`assets`、`cases`、`promptTemplates`。

- `all`：每组默认 5 条，单组最多 8 条。
- 指定分组：默认 30 条，最多 80 条，返回分页信息。
- 关键词压缩连续空白，最长 120 字符。
- 搜索历史使用独立的 `global` 作用域，原有各库历史互不影响。

## 匹配与权限

| 分组 | 匹配字段 | 可见范围 |
| --- | --- | --- |
| 聊天 | 标题、当前用户发送的提示词 | 当前用户未归档、未删除聊天 |
| 图片 | 提示词、类型、尺寸、质量、渠道、时间 | 当前用户图片 |
| 素材 | 名称、来源用户、空间、审核状态、标签 | 自有素材和审核通过的共享素材 |
| 灵感 | 标题、提示词、风格 | 审核通过的灵感，按组去重 |
| 提示词表单 | 名称、描述、分类 | 自有表单和共享表单 |

各组优先展示主标题完全匹配、前缀匹配和包含匹配的记录，再按更新时间排序。服务端不会将“其他用户私有内容”和“不存在内容”区分返回。

## 精确预览

只读详情接口：

```http
GET /api/images/:imageId
GET /api/assets/:assetId
GET /api/cases/:caseId
```

搜索结果使用以下 URL 定位：

```text
/images?keyword=<keyword>&open=<imageId>
/assets?keyword=<keyword>&open=<assetId>
/cases?keyword=<keyword>&open=<caseGroupId>
/prompt-templates?scope=all&keyword=<keyword>&template=<templateId>
/chat/<sessionId>
```

图片、素材和灵感页面会按 `open` 加载目标并打开预览；关闭预览只移除 `open`，保留关键词。目标不存在或权限变化时移除失效参数并显示统一提示。

全局搜索中的“我的图片”结果例外：点击后先读取完整图片详情及其所属会话的全部图片，再直接进入与图片库卡片一致的图片编辑工作台；目标图片保持选中，同会话图片显示在缩略列表中。没有会话的图片只加载自身。素材和灵感结果仍按上述深链接打开预览。

## 首版边界

- 不搜索助手回复、归档聊天、管理后台、提示词生成结果或图片识别文本。
- 不使用 FTS、向量数据库或 AI 语义检索。
- 不改变各库原有独立搜索和普通卡片点击行为。
