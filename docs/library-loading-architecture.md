# 三类图库统一加载架构

## 目标

“我的图片”“素材库”“灵感空间”以及聊天内素材/灵感选择器使用同一套有界加载原则：游标分页、轻量卡片、详情按需读取、独立 facets、缩略图优先和 DOM 虚拟化。业务卡片与操作仍由各页面维护，不把三个领域强行合并成一个组件。

## 接口

主列表接口：

- `GET /api/library/images`
- `GET /api/library/assets`
- `GET /api/library/cases`

请求支持 `limit`、`cursor` 和各领域原有筛选参数。默认 `limit=30`，最大 60。响应统一为：

```json
{
  "items": [],
  "pageInfo": {
    "limit": 30,
    "nextCursor": null,
    "hasMore": false
  }
}
```

游标是带版本、领域、排序、筛选签名、创建时间和稳定 ID 的不透明 Base64URL。客户端不得解析或跨筛选条件复用。服务端使用 `limit + 1` 判断 `hasMore`，不在翻页请求中计算总数。

统计接口独立为 `/api/library/{images|assets|cases}/facets`，服务端缓存 30 秒；同领域写操作成功后清理缓存。灵感额外提供：

- `GET /api/cases/categories`：只返回分类元数据。
- `GET /api/cases/starter?limit=10`：返回有风格分散约束的首页推荐候选。

旧 `/api/images`、`/api/assets`、`/api/cases` 暂时保留兼容，但无参数请求也最多返回 60 条。内部代码不得再依赖无界响应。

## 查询与索引

- 图片和素材使用 `(created_at, id)` 稳定游标。
- 灵感通过 `(group_id, created_at, id)` 反查每组最新可见记录，先按索引和游标取出当前页，再批量补齐封面、组图数、使用数和收藏数；避免窗口函数扫描全部分组。
- 列表只批量补充当前页所需的收藏、标签和封面，不读取参考图或完整组图。
- 详情接口继续负责预览图、原图、参考图和完整组图；页面打开当前项时读取详情，并预取相邻项。
- 图片、素材、灵感都增加了用户/审核/分类与创建时间组合索引；`asset_categories` 增加分类反向索引。

搜索暂时保留原有 Unicode 子串语义。Bun SQLite 虽启用了 FTS5，但默认和 trigram tokenizer 都不能无损覆盖常见的两个汉字查询。本项目在单用户数据超过 5 万条且搜索 p95 超过 300ms 之前不引入语义不一致的 FTS 替代。

## 前端加载

- `useCursorLibraryQuery` 统一下一游标、30 秒 stale time、10 分钟 GC 和 AbortSignal。
- 输入使用 250ms 防抖；筛选变化后旧请求由 TanStack Query 取消。
- `VirtualizedResponsiveGrid` 使用 ResizeObserver、passive scroll、requestAnimationFrame 和二分范围查找，只挂载视口上下约一个视口高度内的行。
- 页面距底部 320px 时预取；选择器以自己的滚动容器作为 observer/virtualization root。
- 第一行缩略图 eager，第一张 high priority，其余 lazy + async decoding。
- 图片编辑器将 facets 总数与当前已加载卡片数分开：总数展示真实筛选结果，游标仍按 30 条续载；在距已加载边界 8 张时预取下一页，侧栏只挂载当前项前后各 8 个缩略图。
- 缩略图继续使用 512px WebP q75；预览使用 1600px WebP q82；图片编辑器主画面始终读取当前选中的原图，相邻导航仍使用缩略图并预取预览资源。文件响应保持 private immutable cache、ETag 和 Cookie vary。
- 响应式宽度动画期间，虚拟网格将同一帧的行高测量合并提交，并在宽度稳定 80ms 后一次性调整列数；成功加载过的缩略图使用有界内存标记避免列数重排后的骨架闪烁。

## 旁路治理

- 聊天页不再初始加载全部素材或灵感。
- 素材选择器打开后使用素材游标接口和服务端筛选。
- 灵感选择器使用灵感游标接口。
- 首页每批只读取 10 条 starter 数据；每日文案与 starter 灵感在父页面并行请求，首批缩略图预解码后统一从骨架状态揭示，失败时分别回退本地文案和默认灵感。自动轮播或手动换组到末尾时请求服务端，并排除当前 10 个分组；新批次预解码完成前继续显示旧批次。
- 新增灵感弹窗只读取分类接口。
- 全局搜索进入图片编辑时，以目标图片为锚点分别读取向新、向旧两个会话游标页；当前图及相邻图按需读取详情，切换到任一边界前继续加载对应方向，不读取完整会话。

## 可观测性与回滚

图库接口返回 `Server-Timing`；超过 200ms 的查询写入慢查询警告。新接口是增量接口，旧接口在迁移期保留，因此可以单独回退前端而不回滚数据库。数据库变化只有索引、`app_migrations` 和一次性预览滚轮默认迁移，没有删除字段。

当前本地真实数据冒烟基线（2026-07-20，热缓存前后会有波动）：30 条素材列表约 6.5KB；30 条灵感列表约 50KB、约 13ms；facets 约 1–5ms。冒烟脚本会遍历全部新旧灵感分页，逐项核对分组 ID、使用/收藏、组图数和下载来源语义，并验证素材中文空间标签搜索。没有登录会话时默认失败，只有显式设置 `SMOKE_ALLOW_SKIP=1` 才允许跳过。

容量基准使用 `bun scripts/benchmark-library-loading.ts`。脚本通过仅用于测试的 `GPT_IMAGE_DATA_DIR`、`GPT_IMAGE_APP_DB_PATH=:memory:` 和 `GPT_IMAGE_CONFIG_DB_PATH=:memory:` 建立隔离数据库，生成图片、素材、灵感各 10 万条，结束后清理临时目录，不接触真实 `data/app.db`。当前机器 10 次热样本结果：

| 接口 | 首屏 p95 | 深游标 p95 | 30 条 JSON |
| --- | ---: | ---: | ---: |
| 图片 | 1.4ms | 20.8ms | 13.7KB |
| 素材 | 129.8ms | 29.9ms | 15.6KB |
| 灵感 | 81.4ms | 96.1ms | 21.2KB |

三类热列表和深游标均低于 250ms，facets 热 p95 为 0.3–2.2ms，卡片 JSON 均低于 150KB。基准现在额外使用不同关键词执行一次不命中缓存的冷 facets，当前结果为：图片约 433ms、素材约 875ms、灵感约 1031ms。冷查询不再被热缓存结果掩盖，但本轮遵守“不引入 FTS/ngram”的边界，不把冷态结果纳入热缓存验收；当真实单用户数据超过 5 万且搜索 p95 持续超过 300ms 时，应启动独立 n-gram 索引阶段。正式环境需要分别观察 `Server-Timing` 的热、冷分布。
