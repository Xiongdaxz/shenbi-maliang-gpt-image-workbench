# 图片批量管理

“我的图片”支持基于图片 ID 的批量选择。选择状态独立于虚拟列表 DOM，因此切换网格、时间线、折叠分组和继续加载分页时不会丢失。

## 交互与限制

- 工具栏进入批量模式，点击卡片切换选择，`Esc` 退出。
- “全选已加载”只选择当前已经加载的图片，不包含未请求的分页结果。
- 选择上限为 200 张；收藏和删除最多 200 张，加入素材最多 100 张，加入灵感最多 20 张，ZIP 下载最多 50 张。
- 批量加入素材时，每张图片根据自己的原始提示词独立生成标签。
- 批量加入灵感时，每张图片独立生成标题和风格，并创建为独立灵感记录。
- 批量模式中隐藏搜索、收藏筛选和排序入口，仍可切换网格与时间线。
- 收藏、加入素材、加入灵感和下载后保留选择；删除后移除已删除图片。
- 批量删除实现暂时保留，但前台操作入口当前隐藏，后续确认交互后再决定是否开放。
- 下载确认弹窗使用带滑动动画的胶囊分段控件切换原图、预览图和缩略图，并分别展示基于图片尺寸和输出规格计算的预计大小。

## 接口

| 接口 | 用途 |
| --- | --- |
| `PUT /api/images/batch/favorite` | 批量收藏或取消收藏 |
| `POST /api/assets/from-images` | 批量加入素材库，重复项跳过 |
| `POST /api/cases/from-images` | 逐图生成标题和风格并创建独立灵感记录 |
| `POST /api/images/batch/delete-preview` | 查询批量删除的关联影响 |
| `POST /api/images/batch/delete` | 确认并执行批量删除 |
| `POST /api/files/images/batch-downloads` | 创建一次性 ZIP 下载令牌 |
| `GET /api/files/images/batch-downloads/:token` | 流式下载 ZIP |

批量操作会对图片 ID 去重并校验当前用户所有权。不存在或不属于当前用户的图片统一按 `not_found` 返回，不暴露其他用户的数据。

## 删除语义

批量删除复用现有单图删除规则。删除图片时还会清理：

- 与图片使用相同文件路径的个人素材；
- 引用这些图片或素材的灵感组；
- 收藏、引用和派生图。

关联聊天消息不会删除。服务端会解除消息的图片引用；助手图片消息显示为“图片已删除”，用户提示词消息继续保留原文。

前端必须先调用删除预览接口。如果存在关联素材、灵感或消息，用户需要额外确认；选择超过 20 张时还需要输入准确数量。服务端执行删除前会重新计算影响，并拒绝未经关联确认的请求。

## ZIP 下载

- ZIP 下载令牌绑定当前用户、仅可使用一次，并在 10 分钟后过期。
- 最多下载 50 张，按原图大小估算总量不得超过 500 MB。
- 支持原图、预览图和缩略图；图片条目使用零压缩，避免重复压缩图片格式。
- 文件名采用序号和图片短 ID，默认附带 UTF-8 `manifest.csv` 与 `manifest.json`。
- 个别文件不可用时继续下载并写入 `_errors.txt`；创建令牌时如果全部文件不可用则直接拒绝。
